OS是什麼?

電腦系統:app,software
lib,system calls,program generation tool

Hardware

Top down View

  • a software handle all hardware resource
  • no needs to worry about low-level things
  • ex: cp memo letter
    • no needs to worry:
    • the location of the file on the hard disk
    • size of disk,brand of disk…

Bottom up View

  • a software (de)allocates all system resource
  • ex:
    • only one file can be printed by a printer
    • preventing from overrides others’ files on disk

OS Services

  • resource manager
  • provides many read-made services

GUI CLI(CUI)
Most OS have both Most OS have both

easy to use flexiablity

user and process categorize

  • Single-user,Single-process
    • one process at a time
    • ex: early Mac OS,DOS,Win
  • Single-user,Multi-process
    • ex: OS/2 Win XP Workstation,batch OS
  • Multi-user,Multi-process
    • ex: Unix,Linux,Win NT(Server),VM
    • increased system throughput
    • multiprogramming

For each type of OS:

  • single-process system:
    • use all power
  • Interactive
    • allow users to interact with their executing prog.
  • Batch os
    • useful when programs are run without human’s instruction
    • Almost all os are interactive

Unix allow programs run in batch mode.

  • Time-sharing system
    • multiuser+multiprocess+interactive

Ch4：file system

  • Unix 把所有東西都視為檔案
  • NIC,hard disk,USB,keboard,printer,傳統檔案…
  • Unix 的七種檔案類型
    • Simple/ordinary file
    • Dir
    • Symbolic link(類似Windows的捷徑)
    • Character special file
    • Block special file
    • Named pipe (FIFO)

  • 強烈建議不要使用空白當檔名

  • 每個檔案都有自己的inode

Link File

  • Points to an exsiting file

Special(or Device) File

  • Used to access hardware devices
  • Each hardware is associated with at least one spcial file
  • two type
    • Charater special files
      • 和字元裝置有關，如鍵盤
    • Block special files
      • 和區塊裝置有關，如硬碟
  • 讓應用程式能以跟存取檔案相同的方法存取裝置
  • 也有一些模擬物理裝置的出現(psedodevices)，讓我們可以從遠端登入到系統(模擬鍵盤、滑鼠等)

Named Pipe(FIFO)

  • Unix 中有一些工具可以讓程序之間溝通

Socket

  • AF_INET : internet domain socket
  • AF_UNIX : same machine under Unix , called Unix domain socket

發表於 2020-01-09 分類於 資料結構

前言

在資料結構中，heap是一種很特別的結構，它可以以O(logn)的時間複雜度插入，並且每次都能以O(1)的時間取出最大或最小的值，再用O(logn)的時間重新調整，如果要求要重複取出最大或最小值時會非常好用！

要怎麼做？

如果提到要取出一個數列中的最大或最小值，大部分的人都會想到要用排序，但就算是最快的排序也需要平均O(nlogn)的複雜度，如果資料一多，這會是個非常恐怖的數字。因此我們換種方法：我們建立一棵樹，讓每一個父結點的值都小於（大於）子結點，那我每次要用的時候就取出最上面的結點，然後再調整一下就好啦！

工具

老樣子，要做出這麼一個東西總得準備一些工具吧！下面列出要用到的資料結構

  1. 陣列一組
  2. 資料一組
  3. 一顆喜樂的心

那我們就開始吧！

補充資料

明明說要用tree做，怎麼用陣列阿？那是因為我們接下用到的tree又被稱為complete binary tree，也就是說，我們的陣列值可以透過 x2 到左子點，x2+1 到右子點，/2就會到父結點，這裡的/只取整數部分喔！陣列從1開始放數值，可以看看下圖：

假設我們要建的heap是Max heap，也就是父結點都大於子結點的heap，然後我們先拿一個現成的heap來示範插入，畢竟只要一個一個插入資料就可以做出一個完整的heap囉！

第一步

先確認好總共有幾個元素在heap中了，把那個數字加一存成heap的新總元素數。像現在這邊已經有5個元素了，就把總數改成6

第二步

從最後一個index除以2往上比。先拿6/2＝3來比，可以看到3這一項現在值是2，比21小，所以我們要把他往下移到6

第三步

從3往上比。3/2＝1，所以去找1有沒有比21大。1的值是20，比21小，所以往下移動到3

第四步

現在的位置已經是21該有的位置了，就把他放進去吧！

至此，我們就已經完成插入了

總結一下

我們用類似程式語言的方式來複習一下

  1. 有一數w插入heap，有一heap目前總數是n，n <- n+1
  2. 如果heap[n]<w，重複以下行為：
    1. heap[n] <- heap[n/2]
    2. n <- n/2
  3. heap[n] <- w

這裡A<-B是指把B的值給A的意思喔！

基因編輯

發表於 2020-01-09 分類於 通識

基因編輯

起源：DNA

  • DNA可說是生命的藍圖
  • 若重要位置發生突變，可能產生重大失調，甚至遺傳疾病
  • 想法：避免產生錯誤的蛋白質，直接編輯錯誤的基因
  • 把錯的地方剪掉，補上新的

剪刀哪裡來？

目前最好最精準的剪刀是CRISPR，來自細菌的免疫系統

  • 1987年，日本科學家在細菌中發現一段奇怪的規律序列，用途不明
  • 過了很久之後，科學家發現這些序列是用來記錄來犯的病毒！！

細菌的病毒？

細菌遇到病毒怎麼可能讓他「打進來」呢？

細菌的防衛機制

  • 細菌在被病毒打進DNA後，細菌會將病毒的DNA剪碎，插入自己的CRISPR內
  • 下次病毒再入侵的時候，細菌會以原本的DNA片段當模板
  • 複製一條RNA給Cas9，去檢查是不是一樣的病毒，是的話就直接剪掉

病毒成功的被細菌「拉進來打出去」

好剪刀，不用嗎

  • 利用跟要剪的部分的互補RNA讓Cas9附著，讓Cas9剪開目標
  • DNA本身就有自動修復的能力，所以只要送入正確的基因就有機會黏在斷口處

好技術就可以發大財啦，我要專利！

美國戰場

  • 原本CRISPR-Cas9的專利是加州大學柏克萊分校的珍妮佛的團隊首先申請（2012/5）
  • 後來哈佛與麻省理工，以張鋒為首的團隊，強調在真核細胞切割（2012/12）
  • 但張鋒利用加速審查制度，先行註冊
  • 但珍妮佛的團隊申請時間適用舊法，即發明人擁有專利
  • 張鋒贏了，珍妮佛的案子仍在審

歐洲戰場

  • 張鋒先申請
  • 但歐洲有9個月的異議時間，目前能在審

張鋒

  • 光遺傳技術

所以這個可以幹嘛？

  1. 古生物復活
    • 目前已有冷凍16年的小鼠復活的例子
    • 猛獁象和亞洲陸象：基因差異只有4%，將差異的部分編輯後即可復活
    • 恐龍：從琥珀中的蚊子血液抽取DNA，尋找代理孕母並編輯基因
  2. 治療疾病
    • 有時候會脫靶（剪錯地方），可能產生無法預期的變異
    • 人類有65億個鹼基序列，但Cas9能辨認的只有23個，統計上至少需要28個，所以也可能找到很像但不一樣的
    • 希望能找到不會有脫靶效應的蛋白
    • 目前比較有可能的作法：
    • 先取出之後修改，定序之後確定沒問題再放回去
    • 可以取出的細胞只有血液中的免疫細胞
    • 美國2019批准
    • 目前所有治療都在一期或二期，沒有副作用但不確定效用
  3. 器官移植
    • 利用體細胞重新分化成多功能分化性幹細胞
    • 對幹細胞基因編輯，相容於某種動物
    • 用動物生產器官，進入人體也不會排斥
  4. 糧食問題
    • 全球暖化和氣候變遷引發植物病蟲害
    • 利用基因編輯技術直接對目標基因做編輯，而不是插入外部基因
    • 抗旱、多產…
  5. 道德爭議
    • 脫靶效應若產出突變物，可能會干擾生態
    • CRISPR的高利潤，使得大部分技術都保密不公開
    • 使用大量農藥等有毒物質，危害環境
    • 對人體未知的危害
    • 復育絕種動物皆為滿足人的慾望，違反實驗動物福祉3R
    • 基因編輯對人體專一，費用龐大，使得醫療更商業化
    • 改造人、改造嬰兒、強化士兵等道德爭議
